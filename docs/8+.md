# 阶段 8+：对话记忆功能实施计划

## Context

当前 RAG 问答（阶段 8）每次提问都是无状态单轮对话。用户无法进行多轮追问，模型无法理解指代词（"它"、"上面提到的"等）。需要加入对话记忆，使问答支持多轮上下文。

## 技术栈

| 类别     | 技术                                                      | 说明                                      |
| -------- | --------------------------------------------------------- | ----------------------------------------- |
| 对话记忆 | LangChain4j `MessageWindowChatMemory` + `ChatMemoryStore` | 滑动窗口记忆，可配置保留轮数              |
| 持久化   | PostgreSQL（新建 2 张表）                                 | 对话元数据 + 消息记录，与现有数据保持一致 |
| 消息窗口 | 可配置 `memory-size`（默认 10 条）                        | 防止上下文过长超出模型限制                |

**不引入额外依赖** —— LangChain4j 0.29.1 已包含 `ChatMemory` / `ChatMemoryStore` / `MessageWindowChatMemory`，无需新增 Maven 依赖。

## 新增数据库表

```sql
-- 对话表
conversations (id, user_id, title, created_at, updated_at)

-- 对话消息表
conversation_messages (id, conversation_id, role, content, created_at)
-- role: 'USER' | 'AI'
```

### 步骤 2：实现 ChatMemoryStore

**新增文件**：
- `assistant/memory/JpaChatMemoryStore.java` — 实现 LangChain4j 的 `ChatMemoryStore` 接口
  - `getMessages(memoryId)` → 从 conversation_messages 按 conversationId 查询
  - `updateMessages(memoryId, messages)` → 全量替换该对话的消息记录

**原理**：将 LangChain4j 的 `memoryId` 映射为 `conversationId`，消息序列化/反序列化用 role + content。

### 步骤 3：配置与参数

**修改文件**：
- `assistant/config/RagConfig.java` — 新增配置项：
  - `memorySize`（默认 10）— 滑动窗口保留的最大消息数
  - `memoryEnabled`（默认 true）— 是否启用对话记忆

- `src/main/resources/application.yml` — rag 配置段添加：
  ```yaml
  rag:
    memory-enabled: true
    memory-size: 10
  ```

### 步骤 5：改造 RagService 集成记忆

**修改文件**：
- `assistant/service/RagService.java`
  - 注入 `JpaChatMemoryStore` 和 `RagConfig`
  - `ask()` 方法改造：
    1. 若 `conversationId` 为空或 `memoryEnabled=false`，保持当前无状态行为
    2. 若有 `conversationId`，创建 `MessageWindowChatMemory` 实例
    3. 将 UserMessage 存入 memory
    4. 构建完整消息列表：`[SystemMessage(RAG上下文)] + chatMemory.messages()`
    5. 调用模型生成回答
    6. 将 AiMessage 存入 memory
  - `askStream()` 同理，在 `onComplete` 回调中存入 AiMessage

- `assistant/dto/AskRequest.java` — 新增可选字段 `conversationId`
- `assistant/dto/AskResponse.java` — 新增字段 `conversationId`

**验证流程**：
1. 创建新对话 → 获得 conversationId
2. 第一次提问"文档中提到了什么？" → 获得回答
3. 第二次追问"能详细说说第一点吗？"（传入相同 conversationId）→ 验证模型理解"第一点"指代
4. 不传 conversationId → 验证回归无状态行为
5. 删除对话 → 验证消息被清除

## 关键设计说明

1. **SystemMessage 不存入记忆**：每轮对话的 SystemMessage 包含实时 RAG 检索结果，不持久化，仅在调用模型时动态构建
2. **滑动窗口**：`MessageWindowChatMemory` 自动保留最近 N 条消息（User+AI），防止上下文过长
3. **向下兼容**：`conversationId` 为可选参数，不传时保持原有的无状态行为
4. **用户隔离**：所有对话操作都校验 userId，确保用户只能访问自己的对话
5. **无新增 Maven 依赖**：LangChain4j 0.29.1 已内置 ChatMemory 所需类
